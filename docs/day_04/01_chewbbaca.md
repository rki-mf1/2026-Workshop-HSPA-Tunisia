# cgMLST with chewBBACA

## 🎯 Learning goals

By the end of this practical, you should be able to:

- understand what cgMLST is used for in bacterial genomic surveillance
- run allele calling with `chewBBACA`
- identify the main output files needed for interpretation and visualization

## Overview

In this practical, we will compare **Klebsiella pneumoniae** genomes using cgMLST.

The general workflow is:

```text
FASTQ files
  ↓
Bactopia
  ↓
Assembly FASTA files
  ↓
chewBBACA allele calling → chewBBACA ComputeMSA → IQ-TREE on chewBBACA-generated MSA
  ↓
cgMLST profile
  ↓
Minimum-spanning tree
```

In the previous practical, we used Bactopia to assemble the FASTQ files. In this practical, we will use the resulting assembly FASTA files as input for chewBBACA.

## 1. Install chewBBACA

```bash
conda create -c bioconda -c conda-forge -n chewie "chewbbaca=3.5.4"
```

---

## 2. Activate chewie conda environment

Activate the environment prepared for this practical.

```bash
conda activate chewie
```

---

## 3. Collect assembly FASTA files and decompress them

Create a directory for the assemblies that will be used by chewBBACA.

```bash
mkdir -p analyses/chewbbaca/assemblies
```

Copy assembly FASTA files into one directory.

```bash
cp analyses/bactopia/bactopia_kp/*/main/assembler/*.fna.gz analyses/chewbbaca/assemblies
```

Move to chewBBACA directory.

```bash
cd analyses/chewbbaca
```

Decompress `.gz` files.

```bash
gunzip -k assemblies/*
```

---

## 4. Creating a schema seed

```bash
chewBBACA.py CreateSchema \
    -i assemblies/ \
    -o kp_schema \
    --ptf ~/2026-Workshop-HSPA-Tunisia/data/prodigal_training_files/Klebsiella_pneumoniae.trn \
    --cpu 2
```

>[!NOTE]
> __Prodigal training files__ `.trn`
> chewBBACA includes Prodigal training files for several species. You can check the list of available training files [here](https://github.com/B-UMMI/chewBBACA/tree/master/CHEWBBACA/prodigal_training_files). The file is also available in `data/prodigal_training_files`.

---

## 5. Determining allelic profiles and adding new alleles to the schema through allele calling

```bash
chewBBACA.py AlleleCall \
    -i assemblies \
    -g kp_schema/schema_seed \
    -o allele_call_out \
    --cpu 2
```

---

## 6. Determining the core genome

```bash
chewBBACA.py ExtractCgMLST \
    -i allele_call_out/results_alleles.tsv \
    -o allele_call_out/cgMLST
```

---

## 7. OPTIONAL | Annotating the schema loci

```bash
chewBBACA.py UniprotFinder \
    -g kp_schema/schema_seed \
    -o schema_annotations \
    -t kp_schema/cds_coordinates.tsv \
    --taxa "Klebsiella pneumoniae" \
    --no-sparql \
    --cpu 2
```

---

## 8. OPTIONAL | Generating an interactive report for schema evaluation

```bash
chewBBACA.py SchemaEvaluator \
    -g kp_schema/schema_seed \
    -o schema_evaluator_out \
    -a schema_annotations/schema_seed_annotations.tsv \
    --loci-reports \
    --add-sequences \
    --cpu 2
```

---

## 9. OPTIONAL | Generating an interactive report to evaluate allele calling results

```bash
chewBBACA.py AlleleCallEvaluator \
    -i allele_call_out \
    -g kp_schema/schema_seed \
    -o allele_call_evaluator_out \
    --a schema_annotations/schema_seed_annotations.tsv \
    --cpu 2
```

## 10. Visualization of Minimum Spanning Trees

You can upload the allelic profiles generated by chewBBACA (`allele_call_out/cgMLST/cgMLST95.tsv`) and associated metadata to software such as [PHYLOViZ Online](https://online2.phyloviz.net/index) to visualize Minimum Spanning Trees (MSTs) and perform various dataset operations that allow you to explore and analyse the results.

It is also possible to use other software such as [GrapeTree](https://github.com/achtman-lab/GrapeTree) to visualize MSTs based on the allelic profiles generated by chewBBACA, but this requires some data formatting to convert the allelic profiles generated by chewBBACA to a format compatible with GrapeTree (more information about the format supported by GrapeTree [here](https://enterobase.readthedocs.io/en/latest/grapetree/grapetree-tutorial-1-sa.html#loading-data-into-grapetree-with-the-stand-alone-version)).

## 11. Interpret the results

Try to answer:

1. How many samples were included in the cgMLST analysis?
2. Did all samples have many cgMLST loci present?
3. Which samples are closest together in the MST?
4. Are any samples separated from the others?
5. Could poor assembly quality explain missing loci?

## 12. Create a multiple sequence alignment for IQ-TREE

The allele calling result can also be converted into a multiple sequence alignment.
This alignment will be used in the next practical with IQ-TREE.

```bash
chewBBACA.py ComputeMSA \
  -i allele_call_out/results_alleles.tsv \
  -g kp_schema/schema_seed \
  -o msa_out \
  --dna-msa \
  --output-variable \
  --cpu 2
```

When the input is a TSV file containing allelic profiles, the output folder structure is as follows:
```bash
msa_out
├── dna_msa.fasta                   (if --dna-msa is used)
├── dna_msa_variable.fasta          (if --output-variable and --dna-msa are used)
├── protein_msa.fasta
├── protein_msa_variable.fasta      (if --output-variable is used)
└── summary_stats.tsv
```

Check the output files.

```bash
ls -lh msa_out
```

Important output files include:

| File                                | Purpose                                        |
|-------------------------------------|------------------------------------------------|
| msa_out/dna_msa.fasta               | Complete DNA alignment from allele calls       |
| msa_out/dna_msa_variable.fasta	    | DNA alignment with variable positions only     |
| msa_out/protein_msa.fasta	          | Complete protein alignment                     |
| msa_out/protein_msa_variable.fasta	| Protein alignment with variable positions only |
| msa_out/summary_stats.tsv	          | Summary of loci included in the MSA            |

For the next IQ-TREE practical, we will use `msa_out/dna_msa_variable.fasta`.
